### Topic: Production API troubleshooting

API at `https://comics-n-stuff-gql-production.up.railway.app/graphql` was returning internal server errors on all data queries while introspection worked. Root cause: Railway (API host) was trying to connect to Supabase (DB host) over IPv6 (`ENETUNREACH` error in logs). Fix: Changed `DATABASE_URL` in Railway from direct connection (port 5432, IPv6) to Supabase connection pooler (port 6543, IPv4). API became fully operational.
